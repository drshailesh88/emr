## Build Configuration Details

### Android (pyproject.toml)

```toml
[tool.flet.android]
package_name = "app.docassist.mobile"
min_sdk_version = 21  # Android 5.0+
target_sdk_version = 34  # Android 14

permissions = [
    "android.permission.INTERNET",
    "android.permission.CAMERA",
    "android.permission.WRITE_EXTERNAL_STORAGE",
    "android.permission.USE_BIOMETRIC",
    "android.permission.VIBRATE",
]
```

## What's Next

### Before First Build

1. **Install prerequisites**:
   - Android: Install Android Studio
   - iOS: Install Xcode (macOS only)

2. **Create placeholder assets**:
   ```bash
   python create_placeholder_assets.py
   ```

3. **Test in desktop mode**:
   ```bash
   python main.py
   # or
   python test_mobile_ui.py
   ```

### First Build

1. **Build debug APK/IPA**:
   ```bash
   ./build_android.sh  # For Android
   # or
   ./build_ios.sh      # For iOS (macOS only)
   ```

2. **Install to device**:
   - Android: `adb install build/apk/DocAssist.apk`
   - iOS: Use Xcode Devices window

3. **Test all features**:
   - Login flow
   - Patient list
   - Sync functionality
   - Offline mode
   - All CRUD operations

### Before Production Release

1. **Replace placeholder assets**:
   - Design professional app icon (1024x1024)
   - Create branded splash screen (2048x2048)
   - See asset README files for guidelines

2. **Configure code signing**:
   - Android: Create keystore for signing
   - iOS: Set up provisioning profiles

3. **Build release version**:
   ```bash
   ./build_android.sh --release
   # or
   ./build_ios.sh --release
   ```

4. **Test thoroughly**:
   - All features on physical devices
   - Different screen sizes
   - Low-end devices
   - Network conditions (offline, slow, fast)
   - Edge cases

5. **Submit to stores**:
   - Google Play Store (Android)
   - Apple App Store (iOS)
   - See BUILD_GUIDE.md for submission process
